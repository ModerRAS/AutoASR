# AutoASR Releases

## v0.6.4 - 2025-11-21

- 📦 **Full/Slim 双版本发布**：Release workflow 同时输出 `auto_asr-windows-full.zip`（随附 `ffmpeg.exe` / `ffprobe.exe`）与 `auto_asr-windows-slim.zip`（纯 `auto_asr.exe`），方便已有 FFmpeg 环境的用户选择更小的下载包。
- 🧩 **VAD 静音补间**：语音活动检测到的静音间隔也会被拆分成补间片段上传，确保整段音频都被识别，避免阈值设置导致的漏句或被割裂的语音。

## v0.6.3 - 2025-11-21

- 📦 **内置 FFmpeg/FFprobe**：GitHub Release 构建会自动从 gyan.dev 下载最新的 FFmpeg 静态包，将 `ffmpeg.exe` 与 `ffprobe.exe` 同捆到 `auto_asr.exe` 所在目录，程序启动时会优先调用同目录下的可执行文件，用户只需解压就能运行，无需额外配置环境。

## v0.6.2 - 2025-11-21

- 🚀 **单次转码**：视频音轨在上传前不再经历“整轨 MP3 -> 片段 MP3”的二次转码，直接按原轨裁剪分段，减少一倍处理时间并避免额外音质损失。
- 🎯 **VAD 直接切轨**：语音活动检测的分段裁剪也基于原始轨道完成，所有分段临时文件在上传后立即清理，磁盘占用更小。

## v0.6.1 - 2025-11-21

- 🛠️ **Release Flow 修复**：修正 GitHub Actions 发布任务写入 `GITHUB_OUTPUT` 的方式，确保新的标签可以正确同步 `RELEASE_NOTES.md` 到 Release 描述。

## v0.6.0 - 2025-11-21

- ♻️ **临时音轨自动清理**：视频音轨只在转写期间生成短暂的 MP3，中间文件在任务结束后即刻删除，避免磁盘被占满。
- 🎬 **SRT 字幕输出**：所有转写结果统一改为 `.srt` 字幕文件，命名方式如 `video.轨道1.srt`；VAD 分段将生成多条字幕条目，非 VAD 模式也会按照整段时长生成合法 SRT。
- 📘 **文档同步**：README / AGENTS 说明更新，突出新的字幕输出和临时文件策略。
- 📝 **Release 自动同步**：GitHub Actions 发布任务会自动提取 `RELEASE_NOTES.md` 中对应版本段落，作为 Release 描述。

## v0.5.0 - 2025-11-21

- 🔤 **界面本地化（中文）**：将主界面、日志、校验提示、README 等用户可见文本统一切换为中文，改善中文用户体验；同时对 GUI 字体做了跨平台兼容处理（微软雅黑 / 苹方 / Noto Sans CJK）。
- 🔁 **即时扫描（Run Now）**：新增“立即扫描”按钮，可在任意时刻立即触发一次目录扫描（会在任务进行中被禁用以避免重复触发）。
- 🛠️ **用户体验改进**：修复日志显示文本、优化 VAD 参数默认值与单位说明，并在 README 中同步新增说明。

## v0.4.0 - 2025-11-20

- ✨ **语音活动检测 (VAD)**：引入 `voice_activity_detector` crate，可在上传前对音频进行语音分段，自动跳过长时间静音并在导出的文本中附带时间戳。可在 GUI 勾选启用，失败时会自动回退至整段上传。

## v0.3.0 - 2025-11-20

- ✅ **结构化实时日志**：Scanner 模块通过异步通道即时推送 `ScanLog`，GUI 不再等待整轮扫描结束即可查看每个文件/音轨的成功与错误信息。
- ✅ **日志显示增强**：主界面日志区域支持等级标签与配色，且最新记录置顶，方便长时间运行时审阅。
- ✅ **中文字体友好**：GUI 统一选择系统自带的 CJK 字体（微软雅黑 / 苹方 / Noto Sans CJK），避免中文字符显示为方块字。
- ✅ **版本升级**：`Cargo.toml` 更新至 `0.3.0`，已通过 `cargo test` 验证。

> 发布说明：推送 `v0.3.0` 标签即可触发 `release.yml` Workflow，在 GitHub Release 中获取打包好的 Windows 版本。
